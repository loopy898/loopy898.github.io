Posted on 2025-08-18 Edited on 2025-08-22

案情背景

于2022年10月，有市民因接获伪冒快递公司的电邮，不慎地于匪徒架设的假网站提供了个人信用咭资料导致经济损失。警方追查下发现当中一名受骗市民男子李大輝 (TaiFai) 的信用卡曾经被匪徒在区内的商舖购物。 后来警方根据IP地址，锁定及拘捕了一名男子林浚熙（阿熙 ChunHei），并于他的居所发现了一批相信曾被用作犯案的电脑及手机装置。 经调查后，警方发现阿熙除上述案件外，他亦牵涉其他的一些犯罪活动。警方的电子数据取证小组在现场作出初步了调查并对涉案装置进行了电子数据取证。请你根据得到的资料，协助将事件经过还原。

准备工作：

下发了加密后的分卷压缩包，先右键-7zip解压并合并（点解压-原目录，不要直接双击，会导致将合并后的结果存在c盘临时文件appdata/tmp/7zip…，且c盘放不下）再用veracrypt磁盘解密，再将解密后的文件放入火眼分析。C:\hlnet这个路径下的文件不要删。

查看容器信息，可见mysql容器内部在监听3306端口。但怎么找对外开放的端口呢？

⭐在Docker中，有两个概念：

  1. 容器端口（Container Port）：容器内部服务监听的端口。
  2. 宿主机端口（Host Port）：将容器端口映射到宿主机上的某个端口，这样外部流量才能通过宿主机访问到容器内部的服务。

docker文件：Docker为每个容器都在 ==/var/lib/docker/containers/== 下创建了一个以容器ID命名的文件夹。这个目录下有两个最重要的文件：

  • hostconfig.json： 这是最重要的文件， 它包含了Docker容器运行时所有的主机配置，其中就包括**端口映射规则。例如，用 docker run -p 43306:3306 这样的命令设置的参数，最终就写在这个文件里。
  • config.v2.json： 包含了容器的其他配置信息。

/var/lib/docker/containers/ 下有两个镜像id（镜像id≠文件夹id），

都打开看一下，

“PortBindings” : {”3306/tcp”: [{”HostIp”:””, “HostPort”:”43306”}]}

意思是将容器内部的 3306/tcp 端口，映射到宿主机（Host）的 43306 端口上。

因此，外部客户端要连接这个MySQL服务，必须使用宿主机的IP地址和 43306 端口;

为什么一个服务器能托管多个网站？

这是 虚拟主机（Virtual Host） 技术的典型应用，通过以下方式实现：

  1. 基于域名区分（最常用）：
    • 所有网站共享同一 IP 和端口（如 80），但通过 server_name 区分请求的域名。
    • 示例：
      • 访问 http://www.kkzjc.com → 由 www.kkzjc.com 配置处理。
      • 访问 http://www.9797jjy.com → 由 www.9797jjy.com 配置处理。
  2. 基于端口区分：
    • 不同网站监听不同端口（如 80、8080、8888），但需用户手动指定端口。
